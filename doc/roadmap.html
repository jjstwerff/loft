Roadmap

Loft is under active development. Everything documented on the language pages works today.

Completed in 0.8.3

Closure capture

Lambdas can now read variables from the surrounding scope. The compiler detects which outer variables the lambda body uses and injects them at call time:

offset = 100;
shift = fn(x: integer) -> integer { x + offset };
assert(shift(5) == 105, "shift(5)");

Note: captured lambdas only work when called directly by name in the same scope. Passing a capturing lambda to map/filter/reduce is not yet supported — pass the extra value as an explicit argument instead.

Tuple types

Return multiple values from a function and destructure them at the call site — no struct definition needed:

fn min_max(lo: integer, hi: integer) -> (integer, integer) {
    (lo, hi)
}

(a, b) = min_max(1, 5);
assert(a == 1 && b == 5, "min_max");

Coroutines / generators

Lazy sequences with yield — generate values on demand without building a collection:

fn count_up(start: integer, stop: integer) -> iterator<integer> {
    for i in start..stop {
        yield i;
    }
}

total = 0;
for n in count_up(1, 6) { total += n; }
assert(total == 15, "sum 1..5");

Use yield from sub_gen() to forward all values from a sub-generator inline. Use next(gen) and exhausted(gen) for manual iteration.

While loop

The while keyword is now supported as syntax sugar over for + break:

i = 0;
while i < 10 { i += 1; }

JSON serialisation and parsing

Any value can be serialised to JSON using the :j format specifier, and parsed back with Type.parse(text):

struct Point { x: float, y: float }
p = Point { x: 1.5, y: 2.5 };
s = "{p:j}";          // {"x":1.5,"y":2.5}
q = Point.parse(s);  // round-trip

No annotations needed — structs, enums, vectors, and primitives all work automatically.

WebAssembly runtime

Loft programs can run entirely in the browser. The compile_and_run() WASM entry point accepts a JSON array of source files, runs the full pipeline in memory, and returns output and diagnostics. A virtual filesystem bridges loft's file-I/O, random, time, and lock APIs to JavaScript host functions. A suite runner in tests/wasm/ validates all docs and scripts against the interpreter.

Native extensions

Rust functions can be registered into loft at runtime using the plugin API:

  • #native "symbol" — annotate a loft function to dispatch to a named Rust symbol
  • cdylib shared libraries loaded at startup via libloading
  • Stable repr(C) LoftPluginCtx ABI published in the loft-plugin-api crate

Language and tooling improvements

  • Exit codes — loft now exits with code 1 on parse or compile errors, making it usable in scripts and CI pipelines.
  • Format specifier errors — using a numeric radix specifier (:x, :b) or zero-padding (:05) on an incompatible type is now a compile error instead of a silent no-op.
  • Generic functions, tuples, and coroutines now compile and run correctly in --native mode.

Still pending for 0.8.3

Cross-scope closures (A5.6)

Same-scope capture (integers, text, mutable) works. One restriction remains: a capturing lambda returned from a function and called from a different scope does not yet work. The fix requires extending Type::Function to carry the closure DbRef alongside the definition number.

WASM threading (W1.18)

par(...) loops are currently sequential in the WASM build. Threading support requires a Web Worker pool, which is planned as a follow-on item.

Coming next

HTTP client

Built-in HTTP requests using ureq:

struct User { name: text, age: integer }

resp = http_get("https://api.example.com/users/1");
if resp.ok() {
    user = User.parse(resp.body);
    println("{user.name} is {user.age}");
}

JSON parsing and serialisation already work today via Type.parse(text) and the :j format specifier. The remaining work is the HTTP transport layer.

Interactive mode

Running loft with no arguments will start an interactive session:

$ loft
> x = 42
> "{x * 2}"
84
> struct Point { x: float, y: float }
> p = Point { x: 1.0, y: 2.0 }
> p.x + p.y
3.0

Definitions persist across lines. A syntax error discards the failed line and continues.

Web IDE

A browser-based IDE that runs the full Loft interpreter as WebAssembly — no installation, no server:

  • Syntax highlighting and error markers (CodeMirror 6)
  • Run button with console output
  • Go-to-definition and find-usages
  • Multiple projects stored locally (IndexedDB)
  • Documentation browser built in
  • Works offline (PWA)

Version 1.0 — stability contract

Version 1.0 means: any program that works on 1.0 will compile and run identically on all future 1.x releases. The language syntax, type system, standard library, and command-line flags are frozen. Until then, breaking changes are possible between minor versions.
