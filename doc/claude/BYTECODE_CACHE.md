# Bytecode Cache (`.loftc`)

> **Status: DEFERRED** — superseded by the Tier N native codegen track.
> Native compilation (0.9.0 target) makes bytecode caching largely redundant.
> This document is retained as a design summary. Tracked as **P4** in ROADMAP.md § Deferred.

Python-style pre-generated bytecode files that allow the interpreter to skip parsing
and compilation entirely when neither the source files nor the loft binary have changed.

---

## What Gets Cached

The complete "runnable state" that must be serialised:

```
State.bytecode        Arc<Vec<u8>>       flat bytecode stream
State.text_code       Arc<Vec<u8>>       long string literals (> 255 bytes)
State.database        Stores             type schema (no runtime data at startup)
fn_positions          Vec<u32>           def_nr → code_position
State.line_numbers    HashMap<u32,u32>   code_pos → source line (debug info only)
```

Not cached (re-established cheaply at load time): native function registry,
stack store, all parser/codegen metadata, all runtime state.

---

## Invalidation Strategy

A cache is valid only when **all** of the following hold:

1. **`format_version`** matches the constant in the current binary.
2. **`binary_hash`** — u64 hash of `std::env::current_exe()`. Any loft upgrade
   automatically invalidates all caches.
3. **All source file mtimes** in the manifest match the filesystem: main script,
   all `default/` files, all `use`-imported library files and their `loft.toml`
   manifests, and (Phase C5) native shared library files.

If any check fails, discard the cache silently and compile normally. On write
failure, emit a non-fatal warning and continue — never abort because of the cache.

**Not checked:** CLI flags, CWD, `LOFT_LIB`. These affect which files are found;
the files themselves are in the manifest and their mtimes are checked.

---

## Cache File Location

**Default:** next to the source file (`script.loft` → `script.loftc`). If the
source directory is not writable, caching is silently skipped.

**`--cache-dir <dir>`:** Stores caches in `<dir>/<hex2>/<stem>_<hex8>.loftc`
where `<hex8>` is a hash of the absolute script path (collision avoidance) and
`<hex2>` is its first two characters (limits directory entry count).
`--cache-dir xdg` resolves to `$XDG_CACHE_HOME/loft` via the `dirs` crate.

---

## CLI Flags

```
--no-cache             Disable all cache read/write. Always compile from source.
--cache-dir <dir>      Store .loftc files in <dir> instead of next to the source.
                       Pass "xdg" for $XDG_CACHE_HOME/loft.
--invalidate-cache     Delete the .loftc file for this script, recompile, write fresh cache.
```

## Phased Rollout

| Phase | Goal |
|-------|------|
| C1 | Minimal working cache for single-file scripts with no `use` imports |
| C2 | `use`-imported library files enter the manifest |
| C3 | Debug info preserved in cache; stripped in production mode |
| C4 | `--cache-dir` / XDG support for read-only source directories |
| C5 | Native extension shared library mtimes enter the manifest (depends on EXTERNAL_LIBS Phase 2) |

Implementation requires:
- New `src/cache.rs` — write/load, `CacheConfig`, source manifest collection, binary hash
- New `src/database/cache.rs` — `Stores::serialize_schema` / `deserialize_schema`
- Changes to `src/main.rs` — read cache before `Parser`; write after `compile::byte_code()`
- Changes to `src/state/mod.rs` — `execute_argv` variant that skips `Data` rebuild

Full implementation details can be derived from the design at implementation time.
Key constraint: `Content::Str` (raw pointer into parse-time source strings) must be
replaced with an owned `String` during serialisation and reconstructed via
`Store::set_str()` on load.
