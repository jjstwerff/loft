# Bytecode Cache (`.loftc`)

Python-style pre-generated bytecode files that allow the interpreter to skip parsing
and compilation entirely when neither the source files nor the loft binary have changed.

## Overview

When a loft script is run and neither its source files nor the loft binary have changed
since the cache was last written, the interpreter loads the `.loftc` cache file and calls
`state.execute_argv("main", ...)` directly — skipping `Parser`, `scopes::check`,
`compile::byte_code`, and all intermediate representation work.

The complete "runnable state" consists of:

```
State.bytecode        Arc<Vec<u8>>            flat bytecode stream
State.text_code       Arc<Vec<u8>>            long string literals (> 255 bytes)
State.database        Stores                  type schema (no runtime data at startup)
fn_positions          Vec<u32>                def_nr → code_position
State.line_numbers    HashMap<u32,u32>        code_pos → source line (debug info only)
```

Fields that are **not** cached, because they are re-established cheaply at load time:

- `State.library` / `library_names` — re-registered by `native::init(state)`.
- `State.stack_cur` — re-created by `State::new()` via `db.database(1000)`.
- All parser/codegen metadata (`stack`, `vars`, `calls`, `types` HashMaps) — not used during execution.
- `Stores.allocations` (the stack store) — created fresh by `State::new()`.
- All runtime state (`files`, `scratch`, `had_fatal`, `logger`, `parallel_ctx`).

## Cache File Format

A `.loftc` file is a single flat binary. All multi-byte integers are little-endian.

### Header (64 bytes, fixed)

| Offset | Size | Field | Notes |
|--------|------|-------|-------|
| 0 | 4 | magic | `0x4C4F4643` ("LOFC") |
| 4 | 2 | format_version | Currently `1`; increment on any breaking layout change |
| 6 | 2 | flags | Bit 0: has_debug_info |
| 8 | 8 | binary_hash | u64 hash of the loft binary (from `current_exe()`) |
| 16 | 4 | num_sources | Number of `(path, mtime)` entries in the source manifest |
| 20 | 4 | bytecode_len | Byte length of the bytecode section |
| 24 | 4 | text_code_len | Byte length of the text_code section |
| 28 | 4 | fn_positions_len | Number of u32 entries in the fn_positions table |
| 32 | 4 | type_schema_len | Byte length of the type schema section |
| 36 | 4 | debug_info_len | Byte length of the debug section (0 if flags bit 0 is clear) |
| 40 | 24 | reserved | Zeroed; reserved for future use |

### Source Manifest

Immediately after the header. `num_sources` records, each:

```
u16   path_len
[u8]  path (UTF-8)
u64   mtime_secs
u64   mtime_nanos
```

The main script is always entry 0. Subsequent entries are default library files
(alphabetical order, matching `parse_dir`) followed by user `use`-imported files.

### Bytecode Section

Raw `Vec<u8>` contents of `State.bytecode`. Loaded directly into `Arc::new(bytes)`.

### Text-Code Section

Raw `Vec<u8>` contents of `State.text_code`. Same treatment.

### Function-Position Table

`fn_positions_len` consecutive `u32` values — the values of
`data.definitions[*].code_position` in definition order. Set directly into
`State.fn_positions` on load, replacing the rebuild that `execute_argv` normally
performs from `Data`.

### Type Schema Section

Encodes `Stores.types: Vec<Type>` and `Stores.names: HashMap<String,u16>`.
See [What Must Be Serialized](#what-must-be-serialized) for the encoding.

### Debug Info Section (optional)

Only present when flags bit 0 is set:

```
u32               line_numbers_count
(u32, u32) * N    (code_pos, line_nr) pairs — State.line_numbers
u32               source_names_count
for each entry:
    u16   source_nr
    u16   path_len
    [u8]  path (UTF-8)
```

## Invalidation Strategy

A cache is valid only when **all** of the following hold:

1. **`format_version`** matches the constant in the current binary.
2. **`binary_hash`** matches a freshly computed hash of `std::env::current_exe()`. Computed
   once at startup and reused. Any loft upgrade automatically invalidates all caches.
3. **All source file mtimes** in the manifest match the current filesystem metadata,
   using both seconds and nanoseconds. This covers the main script, all `default/` files,
   all `use`-imported library files, their `loft.toml` manifests, and (Phase C5) native
   shared library files.

If any check fails, the cache is silently discarded and normal compilation proceeds. After
successful compilation the cache is rewritten. On any I/O error during a write, emit a
non-fatal warning and continue; never abort a run because the cache could not be written.

**What is not checked:** CLI flags (`--path`, `--lib`, `--project`), CWD, `LOFT_LIB`.
These affect which files are *found*; the files themselves are in the manifest and their
mtimes are checked.

## Cache File Location

### Default: next to the source file

```
/path/to/script.loft   →   /path/to/script.loftc
```

If the source directory is not writable and `--cache-dir` is not set, caching is silently
skipped. No error, no warning — execution proceeds normally without a cache.

### `--cache-dir <dir>` override

```
<dir>/<hex2>/<stem>_<hex8>.loftc
```

where `<hex8>` is an 8-character hex digest of the absolute script path (collision
avoidance) and `<hex2>` is its first two characters (limits directory entry count).

`--cache-dir xdg` resolves to `dirs::cache_dir().unwrap().join("loft")` using the `dirs`
crate already in `Cargo.toml`.

Add `*.loftc` to the project's `.gitignore` (documented, not enforced).

## What Must Be Serialized

### Type Schema Encoding

`Stores.types` contains `Type` structs. The `Parts` enum has 13 variants; `Field` structs
contain `Content::Str` (a raw pointer into parse-time source strings) which is **not
serializable**. The `Content::Str` is replaced with an owned `String` during serialization
and reconstructed via `Store::set_str()` on load.

Because `parts` and `fields` use `pub(super)` visibility, serialization must be exposed
as a method on `Stores`: `pub fn serialize_schema(&self, out: &mut Vec<u8>)` and
`pub fn deserialize_schema(bytes: &[u8]) -> Stores`. Both live in a new
`src/database/cache.rs` submodule inside the `database` module.

Wire encoding:

```
u32   type_count
for each Type:
    u16   name_len, [u8] name
    u8    parts_tag      (0=Base 1=Struct 2=Enum 3=EnumValue 4=Byte 5=Short
                          6=Vector 7=Array 8=Sorted 9=Ordered 10=Hash 11=Index 12=Spacial)
    [variant-specific fields — counts and indices as u16/u32]
    u32   keys_count     (each: u8 field_nr, u8 ascending)
    u32   parents_count, u16 parents[N]
    u8    complex, u8 linked
    u16   size, u8 align

u32   names_count
for each (name, index):
    u16   name_len, [u8] name, u16 index
```

`Field` encoding (inside Struct / EnumValue variants):

```
u16   name_len, [u8] name
u16   content        (type index)
u16   position
u8    default_tag    (0=Long 1=Float 2=Single 3=Str)
[8 bytes for Long/Float, 4 for Single, u16+bytes for Str]
u32   other_indexes_count, u16 other_indexes[N]
```

The 7 base types registered by `Stores::new()` are always present at indices 0–6 and are
included in the serialization like any other type (their `parts_tag = 0`, Base). The
deserializer calls `Stores::new()` first to get the pre-populated base types, then
overwrites all entries from the cache.

## Write Path

Inserted in `main.rs` immediately after `compile::byte_code()` succeeds:

```rust
// Pseudocode — main.rs
if !cache_config.no_cache {
    if let Some(cache_path) = cache_config.resolve_path(&abs_file) {
        let manifest = cache::collect_manifest(&p, &abs_file, &dir);
        match cache::write(&cache_path, &state, &p.data, &manifest, &cache_config) {
            Ok(_) => {}
            Err(e) => eprintln!("loft: warning: could not write cache: {e}"),
        }
    }
}
```

**Atomicity:** Write to `<path>.tmp`, then `std::fs::rename` over the target. Rename is
atomic on POSIX; prevents a half-written cache from being treated as valid.

### Collecting the Source Manifest

Two sources are combined:

1. **`default/` files** — walk the directory tree in alphabetical order (same as
   `parse_dir`), collect all `.loft` paths and their mtimes.
2. **User `use`-imported files** — a new `pub imported_sources: Vec<String>` field on
   `Parser` (populated in `parse_file()` at each `lib_path()` call site alongside the
   existing `todo_files.push`) collects every resolved library path.

The main script's absolute path is always entry 0.

## Read Path

In `main.rs`, before constructing `Parser`:

```rust
// Pseudocode — main.rs
let binary_hash = cache::compute_binary_hash();  // hash current_exe(), memoized

if !cache_config.no_cache {
    if let Some(cache_path) = cache_config.resolve_path(&abs_file) {
        match cache::try_load(&cache_path, &abs_file, binary_hash) {
            Ok(Some(loaded)) => {
                // Fast path: no parse, no compile
                let mut state = State::new(loaded.stores);
                state.bytecode  = Arc::new(loaded.bytecode);
                state.text_code = Arc::new(loaded.text_code);
                state.fn_positions = loaded.fn_positions;
                if let Some(dbg) = loaded.debug_info {
                    state.line_numbers = dbg.line_numbers;
                }
                native::init(&mut state);
                // ... logger setup, execute_argv_cached, exit
            }
            Ok(None) => {}   // stale or absent — fall through to normal compile
            Err(e) => {
                if cache_config.verbose { eprintln!("loft: cache load failed: {e}"); }
                // fall through
            }
        }
    }
}
// Slow path: full parse + compile + write cache
```

`try_load` returns `Ok(None)` for any invalidation failure (stale mtime, version mismatch,
bad magic). Only true I/O errors produce `Err`.

`execute_argv` currently rebuilds `fn_positions` from `data.definitions` at startup. For
the cached path, add an `execute_argv_from_cache` variant (or make the rebuild conditional
on `fn_positions` being empty) so no `Data` is required.

`LoadedCache` struct:

```rust
pub struct LoadedCache {
    pub bytecode:      Vec<u8>,
    pub text_code:     Vec<u8>,
    pub fn_positions:  Vec<u32>,
    pub stores:        Stores,
    pub debug_info:    Option<DebugInfo>,
}

pub struct DebugInfo {
    pub line_numbers:  HashMap<u32, u32>,
    pub source_names:  HashMap<u16, String>,
}
```

## Interaction with Stores::write_data / read_data

The existing `Stores::write_data` and `read_data` in `src/database/io.rs` are **not
reusable** for cache serialization. They serialize *instances* (one record's field values
into a live `Store`), not the *type schema*. `read_data` even allocates new vector records
at runtime via `vector::vector_append`.

The cache needs to serialize the schema metadata itself (`Stores.types`, `Stores.names`).
This is an entirely different concern — a **separate serialization path in
`src/database/cache.rs` is required**.

Importantly, the `Store` data inside `Stores.allocations` does **not** need to be cached.
At startup, the only allocation is the stack store created by `db.database(1000)` inside
`State::new()`. User data (records, vectors, hash tables) is populated at runtime by the
bytecode, not at compile time.

## Debug Info

Debug info is preserved in the cache but optional:

- **Development builds** (no `--production`): include debug info (flags bit 0 set).
- **Production builds** (`--production`): strip debug info to reduce file size.
- A future `--strip-debug` flag overrides the heuristic.

The debug section stores `State.line_numbers` (populated by `Value::Line(line)` emission
in `src/state/codegen.rs`) and a source-index → file-path table. After a cache load,
these are restored onto `State` so that error messages still include file names and line
numbers.

## CLI Flags

Added to `main.rs` and `print_help()`:

```
--no-cache             Disable all cache read and write. Always compile from source.
--cache-dir <dir>      Store .loftc files in <dir> instead of next to the source.
                       Pass "xdg" for $XDG_CACHE_HOME/loft.
--invalidate-cache     Delete the .loftc file for this script, recompile, write fresh cache.
```

Stored in a `CacheConfig` struct parsed alongside the existing flags:

```rust
struct CacheConfig {
    no_cache:  bool,
    cache_dir: Option<CacheDirSpec>,   // None = next to source
    invalidate: bool,
    verbose:    bool,                  // print cache hit/miss to stderr
}
```

`--invalidate-cache` deletes the cache file first, then proceeds as normal (compiles and
writes a fresh cache).

## Interaction with External Libraries

### Pure-loft libraries (`use mylib;`)

The resolved `.loft` file path is added to `Parser.imported_sources` and enters the
manifest exactly like any other file. The `loft.toml` manifest file is also included.

### Native extension libraries (EXTERNAL_LIBS Phase 2)

The `Parser.pending_native_libs` list (resolved `.so`/`.dylib`/`.dll` paths) is read by
`collect_manifest()` and their mtimes are added to the manifest. A changed shared library
automatically invalidates the cache.

### Source manifest — complete list

| Source | Collected from |
|--------|---------------|
| Main script | `abs_file` in `main.rs` (entry 0) |
| `default/` `.loft` files | Walk `dir + "default"` (alphabetical, same as `parse_dir`) |
| `use`-imported `.loft` files | `Parser.imported_sources` (new field) |
| Library `loft.toml` manifests | Alongside the library `.loft` file (EXTERNAL_LIBS Phase 1) |
| Native shared libraries | `Parser.pending_native_libs` (EXTERNAL_LIBS Phase 2) |

## Phased Rollout

### Phase C1 — Minimal Working Cache

**Goal:** Cache works for single-file scripts with no `use` imports.

**Changes:**

1. `src/cache.rs` (new) — `write()`, `try_load()`, `CacheConfig`, `LoadedCache`,
   `collect_manifest()` (main script only), `compute_binary_hash()`.
2. `src/database/cache.rs` (new, inside `database` module) — `Stores::serialize_schema()`
   and `Stores::deserialize_schema()`.
3. `src/main.rs` — read cache before `Parser` construction; write cache after `byte_code()`;
   parse `--no-cache`, `--cache-dir`, `--invalidate-cache`.
4. `src/state/mod.rs` — add `execute_argv_from_cache()` (or conditional `fn_positions`
   rebuild) to run without a `Data` argument.

No changes to `src/compile.rs`, `src/parser/`, `src/scopes.rs`.

**Acceptance:** Second invocation of an unchanged script uses only file open/read; no
parse or compile activity.

---

### Phase C2 — Library File Invalidation

**Goal:** `use`-imported library files enter the manifest and trigger invalidation.

**Changes:**

1. `src/parser/mod.rs` — add `pub imported_sources: Vec<String>` to `Parser`; populate
   it in `parse_file()` at the `lib_path()` call site alongside `todo_files.push`.
2. `src/cache.rs` — `collect_manifest()` reads `p.imported_sources` and walks `default/`.

---

### Phase C3 — Debug Info and Production Strip

**Goal:** Error messages work correctly after a cache load; production caches are smaller.

**Changes:**

1. `src/cache.rs` — write/read debug section (flags bit 0).
2. `src/main.rs` — pass `production` flag into `CacheConfig`.
3. `src/state/debug.rs` — restore `line_numbers` from `LoadedCache.debug_info`.

---

### Phase C4 — XDG Cache Dir and `--cache-dir`

**Goal:** Configurable cache location for read-only script directories.

**Changes:**

1. `src/cache.rs` — collision-safe `<dir>/<hex2>/<stem>_<hex8>.loftc` path construction.
2. `src/main.rs` — parse `--cache-dir xdg` and resolve via `dirs::cache_dir()`.

---

### Phase C5 — Native Extension Integration

**Depends on:** EXTERNAL_LIBS Phase 2.

**Goal:** Native shared library mtimes enter the manifest.

**Changes:**

1. `src/cache.rs` — `collect_manifest()` reads `p.pending_native_libs`.
