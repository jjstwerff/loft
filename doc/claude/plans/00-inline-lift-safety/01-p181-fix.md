## Goal

Apply the Option B gate from Phase 0: `OpCopyRecord`'s `0x8000`
free-source flag should only be set when the callee's return
type has an EMPTY `dep` chain (i.e. the return is an owned fresh
store, not a view into one of the args).

## Exact change

`src/state/codegen.rs::gen_set_first_ref_call_copy` at lines
1280–1290.  Replace:

```rust
#[cfg(not(feature = "wasm"))]
let tp_with_free = i32::from(tp_nr) | 0x8000;
#[cfg(feature = "wasm")]
let tp_with_free = i32::from(tp_nr);
```

With:

```rust
// P181: the 0x8000 "free source after deep copy" flag is only
// safe when the callee returns an OWNED fresh store.  When the
// return carries a `dep` chain, the result is a borrowed view
// into one of the args (e.g. `vec[i]` returned from
// `fn first(v: vector<T>) -> T`), and freeing the "source"
// corrupts the arg's store.  Return-dep inference already tags
// these accurately — consult it before setting the flag.
let returned = stack.data.def(*fn_nr_for_returned).returned.clone();
let is_borrowed_view = !returned.depend().is_empty();
#[cfg(not(feature = "wasm"))]
let tp_with_free = if is_borrowed_view {
    i32::from(tp_nr)
} else {
    i32::from(tp_nr) | 0x8000
};
#[cfg(feature = "wasm")]
let tp_with_free = i32::from(tp_nr);
```

`fn_nr_for_returned` is the callee's def_nr.  It's available via
the `value` argument: `gen_set_first_ref_call_copy` is called
with `value = Value::Call(fn_nr, args)`; extract `fn_nr` at the
top of the function (or inline within the new check).  The
existing `value.clone()` at line 1287 already uses it — mirror
that extraction.

## Non-negotiable invariants to preserve

1. **Issue #120 leak regression must stay fixed.**  The `0x8000`
   flag was added to prevent leaks when inline-lift creates a
   temp and the callee returned a fresh store.  After the gate,
   owned-return callees still set the flag, so issue #120 stays
   fixed.  If any `tests/lib/issue_120_*.loft` exists (Phase 0
   didn't audit — do it here), confirm it still passes.
2. **No new opcodes.**  Gate-only; no ABI or runtime changes.
3. **Existing P179 regression tests must stay green.**  The P179
   fix introduced work-refs for non-Var `&T` args; the P181
   gate is upstream of that machinery and shouldn't interact.

## Risks & mitigations

- **The gate's data lookup**: `stack.data.def(fn_nr).returned`
  must be set by the time codegen runs.  The existing
  `gen_set_first_ref_call_copy` already reads
  `stack.data.def(d_nr).known_type` at line 1244, so definitions
  are populated.  `returned` should be too.  Sanity-check by
  building a trivial loft program before running the fixtures.
- **A callee with missing dep annotation**: if ANY accessor-style
  callee in the existing codebase fails to carry its dep, the
  gate misses it and the corruption still fires there.  The
  deferred variants (step 4) are specifically designed to catch
  these.  If any fails post-fix, Phase 1 splits: the gate
  lands, and a follow-up phase patches the dep-inference hole.
- **WASM path**: `cfg(feature = "wasm")` already disables the
  flag entirely.  The gate code only needs to apply to the
  non-WASM branch — no change to WASM behaviour.
